# Отсутствующие метрики Managed Kafka в VictoriaMetrics

> Сгенерировано автоматически на основе сравнения с документацией Yandex Cloud.
> Файл содержит метрики, которые описаны в спецификации, но не поступают из API мониторинга.

## Kafka-специфичные метрики (не поступают из API Yandex Cloud)

| Имя метрики | Описание | Метки |
|-------------|----------|-------|
| `kafka_controller_ControllerStats_LeaderElectionRateAndTimeMs` | Смена брокера-лидера за единицу времени, в норме показывает `0`. При проведении технического обслуживания может расти, что не является тревожным сигналом. | `quantile` |
| `kafka_server_ZooKeeperClientMetrics_ZooKeeperRequestLatencyMs` | Задержка запросов в ZooKeeper. | `quantile` |

## Дисковые метрики квот и гистограммы (не поступают из API Yandex Cloud)

| Имя метрики | Описание | Единицы |
|-------------|----------|---------|
| `io_quota_utilization_percentage` | Средний процент используемой квоты диска. | % |
| `io_quota_utilization_percentage_burst` | Максимальный процент используемой квоты диска. | % |
| `read_bytes` | Среднее количество байт, прочитанных с диска. | байты/с |
| `read_bytes_burst` | Максимальное количество байт, прочитанных с диска. | байты/с |
| `read_latency` | Гистограмма распределения времени обработки запросов на чтение с диска. | миллисекунды |
| `read_ops` | Среднее количество операций чтения с диска. | операции/с |
| `read_ops_burst` | Максимальное количество операций чтения с диска. | операции/с |
| `read_throttler_delay` | Гистограмма задержки, вносимой в длительность выполнения операций чтения при превышении квоты диска. | миллисекунды |
| `write_bytes` | Среднее количество байт, записанных на диск. | байты/с |
| `write_bytes_burst` | Максимальное количество байт, записанных на диск. | байты/с |
| `write_latency` | Гистограмма распределения времени обработки запросов на запись на диск. | миллисекунды |
| `write_ops` | Среднее количество операций записи на диск. | операции/с |
| `write_ops_burst` | Максимальное количество операций записи на диск. | операции/с |
| `write_throttler_delay` | Гистограмма задержки, вносимой в длительность выполнения операций записи при превышении квоты диска. | миллисекунды |

## Примечание

Если эти метрики понадобятся в будущем, возможно:
1. Yandex Cloud Monitoring API начнет их отдавать в Prometheus endpoint (`/monitoring/v2/prometheusMetrics`) — тогда они появятся автоматически благодаря `VMStaticScrape`.
2. Потребуется настройка дополнительных правил сбора через `yc monitoring metrics list` и custom exporter.
